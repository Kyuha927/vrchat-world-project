# 윈도우 환경 이관 안내서 (Windows Handover)

이 파일은 Mac 환경에서 네트워크 차단 이슈로 인해 윈도우 환경으로 개발을 이관하면서, 다음 AI 에이전트(또는 현재 세션)가 상황을 바로 파악하고 작업을 이어갈 수 있도록 작성된 프롬프트입니다.

## 🎯 현재까지 완료된 작업 (Mac)
- **모바일 플레이 대응 완료**: 터치 패드 UI(`web-belltui/game.js`), 반응형 크기 조절, 터치 이벤트 맵핑.
- **레벨 디자인 개선**: 6층 구조의 난이도 스케일링, 고정 1200px 월드 너비, 90초 타이머, 층별 특화 이웃 배치.
- **네트워크 설정**: `web-belltui/server/index.js`에 로컬 IP 출력 기능 추가 (단, Mac 환경의 EPERM 이슈로 테스트하지 못함).

## 🚀 윈도우에서 이어서 할 작업 (Prompt)

윈도우에서 프로젝트를 clone/pull 한 뒤, 아래 프롬프트를 복사해서 에이전트에게 전달하세요.

---
**[복사할 프롬프트 시작]**
현재 '벨튀 대작전' 웹 버전을 윈도우로 가져왔어.
다음 작업을 순서대로 진행해줘:

1. **서버 실행 및 모바일 테스트**
   - `web-belltui/server` 폴더에서 `npm install` 후 `node index.js` 실행
   - 서버가 `0.0.0.0` 또는 `::` 에 정상적으로 바인딩되는지 확인
   - 출력된 로컬 IP(예: `192.168.x.x:3000`)를 통해 스마트폰에서 접속해보고 모바일 터치 컨트롤(좌우 이동, 위/아래층, 대시, 종치기)이 잘 작동하는지 테스트해줘.

2. **VRChat Unity 연동 (UdonSharp)**
   - 이제 웹 버전에 구현된 로직(층별 난이도, 90초 타이머, 악명 시스템)을 Unity 3D `GameManager.cs` 와 `DoorController.cs`에 똑같이 이식해야 해.
   - `world_spec/`의 구조를 바탕으로 에셋들을 배치해줘.

**[복사할 프롬프트 끝]**
