# 🔔 쵸로키의 벨튀 대작전 — 게임모드 기획서

> **두 가지 핵심 모드**: 시청자 참여형 / 스트리머 대전형

---

## 🅰 모드 1: 시청자 참여형 — "우리 동네를 지켜라"

> **컨셉**: 스트리머(쵸로키)가 벨튀를 하고, **시청자들이 이웃 주민이 되어** 막는다.

### 역할 분배

| 역할 | 누가 | 조작 방식 |
|:---|:---|:---|
| **빌런 (쵸로키)** | 스트리머 1명 | 키보드/게임패드 직접 조작 |
| **주민들** | 시청자 전원 | 채팅 명령어로 참여 |

### 시청자 참여 메커니즘 (채팅 연동)

시청자는 **트위치/SOOP 채팅**으로 실시간 참여:

| 채팅 명령어 | 효과 | 쿨타임 |
|:---|:---|:---|
| `!문열어` | 자기가 배정된 문을 즉시 열고 나감 | 15초 |
| `!함정` | 자기 문 앞에 바나나 껍질 설치 (스트리머 감속) | 30초 |
| `!강아지` | 강아지를 풀어서 복도 순찰 | 45초 |
| `!경보` | 해당 층 전체에 경보 발동 (3초간 모든 문 열림) | 60초 |
| `!응원` | 빌런에게 스피드 부스트 (배신!) | 20초 |
| `!CCTV` | 빌런 위치를 3초간 모든 주민에게 표시 | 25초 |

### 시청자 배정 시스템

1. **게임 시작 시** 채팅에 `!참여`를 친 시청자에게 **랜덤으로 문(방) 배정**
2. 각 시청자에게 **이웃 캐릭터** 랜덤 부여 (할머니, 근육맨, 유령 등)
3. 시청자 닉네임이 **문 위에 표시**됨 → "OOO님의 집"
4. 자기 문 앞에서 벨튀 당하면 **시청자 리액션 이모트** 자동 표시
5. 벨튀 성공 시 해당 시청자에게 "당했다!" 뱃지 부여

### 라운드 구조

```
[준비] 시청자 참여 접수 (30초)
  ↓
[라운드 1~3] 빌런이 벨튀 시도 (각 60초)
  - 라운드마다 층 추가 (1F → 2F → 3F)
  - 시청자들은 채팅으로 실시간 방어
  ↓
[보스 라운드] 경비실장 (특수 시청자 or AI)
  - 전 층 순찰, 함정 다수 배치
  - 생존하면 빌런 승리
  ↓
[결과] 점수 발표 + MVP 시청자 선정
```

### 점수 체계

| 대상 | 점수 조건 |
|:---|:---|
| **빌런** | 벨튀 1회 성공 = +100, 콤보 보너스, 전층 클리어 보너스 |
| **시청자(주민)** | 빌런 잡기 = +200, 함정으로 감속 = +50, 경보 발동 = +30 |
| **MVP** | 빌런을 가장 많이 잡은 시청자 또는 가장 많이 벨튀 당한 시청자(불명예) |

### 특수 이벤트 (랜덤 발동)

| 이벤트 | 효과 |
|:---|:---|
| **정전** | 5초간 화면 어두워짐. 빌런에게 유리 |
| **택배 도착** | 모든 문이 동시에 열림. 혼란의 카오스 |
| **고양이 탈출** | 복도에 고양이 등장. 닿으면 3초 스턴 |
| **엘리베이터 고장** | 층 이동 불가 5초 |
| **빌런 타임** | 빌런 스피드 2배 + 투명화 3초 |

## 🅱 모드 2: 스트리머 대전 — "빌런 vs 빌런"

> **컨셉**: 스트리머 2~4명이 **동시에 같은 아파트**에서 벨튀 경쟁. 서로 방해도 가능.

### 기본 규칙

- **플레이어**: 2~4명의 스트리머
- **목표**: 제한 시간 내에 가장 많은 벨튀 성공
- **핵심**: 같은 문을 노리는 경쟁 + 서로 방해 가능

### 캐릭터 선택

각 스트리머는 **빌런 캐릭터**를 선택 (능력치 차이):

| 빌런 | 특수 능력 | 약점 |
|:---|:---|:---|
| **쵸로키 (해머)** | 해머로 문 강제 오픈 (벨 안 눌러도 됨) | 소리가 커서 주민이 빨리 나옴 |
| **스피드런너** | 이동속도 최고, 대시 쿨타임 없음 | 체력 1 (한 번 잡히면 탈락) |
| **유령형** | 5초 투명화 가능, 주민 감지 범위 축소 | 벨 누르는 시간이 2배 |
| **함정사** | 바나나, 압정 등 함정 설치 가능 (다른 빌런 방해) | 이동속도 느림 |

### 대전 모드 종류

#### 🏆 모드 A: 프리포올 (Free-for-All)
- 2~4명 개인전
- 3분 × 3라운드
- 같은 아파트에서 동시 플레이
- **서로 밀치기 가능** (상대를 주민 앞으로 밀어서 잡히게 하기)
- 최다 벨튀 성공자가 승리

#### 🤝 모드 B: 팀전 (2vs2)
- **빌런 팀 (2명)** vs **주민 팀 (2명)**
- 빌런 팀: 협동하여 벨튀 목표 달성
- 주민 팀: 직접 주민을 조작하여 빌런 잡기
- 역할 스왑 후 점수 비교

#### 👑 모드 C: 왕좌 쟁탈전
- 라운드마다 **최하위 탈락**
- 5명 시작 → 4 → 3 → 2 → 최종 1:1
- 마지막 라운드는 **보스 아파트** (주민 AI 최강)

### 대전 맵 종류

| 맵 이름 | 특징 | 난이도 |
|:---|:---|:---|
| **평범한 아파트** | 5층, 문 적음, 숨는 곳 많음 | ★☆☆ |
| **고급 빌라** | 3층, 문 넓게 배치, CCTV 있음 | ★★☆ |
| **미로 아파트** | 7층, 복도가 꺾이고 막다른 길 있음 | ★★★ |
| **유령 아파트** | 4층, 조명 깜빡임, 유령 주민 다수 | ★★★ |
| **쵸로키의 아지트** | VRChat 맵 연동, 빌런 테마 | ★★☆ |

### 아이템 시스템 (대전 전용)

복도에 랜덤 스폰되는 아이템:

| 아이템 | 효과 |
|:---|:---|
| 🍌 **바나나** | 설치 → 다른 빌런이 밟으면 3초 슬립 |
| 👟 **운동화** | 10초간 스피드 1.5배 |
| 👻 **유령 망토** | 5초 투명화 (주민에게 안 보임) |
| 📢 **확성기** | 다른 빌런 주변의 주민을 전부 깨움 (방해) |
| 🔔 **황금 벨** | 한 번에 3개 문 동시 벨튀 |
| 🛡️ **방패** | 주민에게 한 번 잡혀도 면역 |
| 🐱 **고양이** | 던지면 대상 3초 스턴 |

## 🔧 기술 구현 방향

### 시청자 참여형
| 항목 | 기술 |
|:---|:---|
| 채팅 연동 | Twitch/SOOP Chat API (WebSocket) |
| 시청자 명령 처리 | Node.js 서버 or 브라우저 내 tmi.js |
| 화면 표시 | OBS 브라우저 소스 (현재 HTML5 게임 확장) |
| 시청자 배정 | 서버 사이드 세션 관리 |

### 스트리머 대전
| 항목 | 기술 |
|:---|:---|
| 멀티플레이 | WebSocket 실시간 동기화 |
| 서버 | Node.js + Socket.io (또는 Colyseus) |
| 매칭/로비 | 방 코드 입력 방식 (간단) |
| 맵 동기화 | 서버 권위형 (Server-authoritative) |

### 공통
| 항목 | 기술 |
|:---|:---|
| 렌더링 | HTML5 Canvas (현재 베이스 활용) |
| 사운드 | Web Audio API + 이펙트 샘플 |
| 배포 | GitHub Pages 또는 자체 서버 |
